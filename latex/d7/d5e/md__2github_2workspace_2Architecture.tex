This project is an embedded application, designed to run on an {\ttfamily n\+RF51} microcontroller with an {\ttfamily arm-\/cortex-\/m0} CPU. The architecture is seperated into three main layers; the hardware abstraction layer, the logic layer and the presentation layer. Additionally modules for the entry process and utility functions exist. This encapsulation allows for a flexible design.\hypertarget{md__2github_2workspace_2Architecture_autotoc_md5}{}\doxysection{\texorpdfstring{🚪 Entry Module}{🚪 Entry Module}}\label{md__2github_2workspace_2Architecture_autotoc_md5}
{\itshape files\+: \doxylink{entry_8S}{entry/entry.\+S}, entry/linker\+\_\+script.\+ls}

This module is responsible for the startup process of the microcontroller. Here important elements, such as the entry point, interrupt vector table and memory regions are defined and actions such as copying data from ROM to RAM are performed.\hypertarget{md__2github_2workspace_2Architecture_autotoc_md6}{}\doxysection{\texorpdfstring{📟 Hardware Abstraction Layer}{📟 Hardware Abstraction Layer}}\label{md__2github_2workspace_2Architecture_autotoc_md6}
{\itshape files\+: include/hal/\texorpdfstring{$\ast$}{*}.h, src/hal/\texorpdfstring{$\ast$}{*}.c}

The hardware abstraction layer, short hal, abstracts low level operations such as register access and encapsulates these operations, allowing the layers above to refrain from interacting with any of the peripherals directly.\hypertarget{md__2github_2workspace_2Architecture_autotoc_md7}{}\doxysection{\texorpdfstring{🧩 Logic Layer}{🧩 Logic Layer}}\label{md__2github_2workspace_2Architecture_autotoc_md7}
{\itshape files\+: include/logic/\texorpdfstring{$\ast$}{*}.h, src/logic/\texorpdfstring{$\ast$}{*}.c}

Within this layer, the game logic itself resides. Here you can find the main game loop, the timer and the game state.\hypertarget{md__2github_2workspace_2Architecture_autotoc_md8}{}\doxysection{\texorpdfstring{🖥️ Presentation Layer}{🖥️ Presentation Layer}}\label{md__2github_2workspace_2Architecture_autotoc_md8}
{\itshape files\+: include/presentation/\texorpdfstring{$\ast$}{*}.h, src/presentation/\texorpdfstring{$\ast$}{*}.c}

This layer contains everything used to display a pretty output to the user. Menus, ui and styling can all be found here.

The seperation from the logic layer allows for flexible configuration, such as changing field sizes during the game.\hypertarget{md__2github_2workspace_2Architecture_autotoc_md9}{}\doxysection{\texorpdfstring{🛠️ Utility Layer}{🛠️ Utility Layer}}\label{md__2github_2workspace_2Architecture_autotoc_md9}
{\itshape files\+: include/util/\texorpdfstring{$\ast$}{*}.h, src/util/\texorpdfstring{$\ast$}{*}.c}

Modules used throughout the layer, or which can\textquotesingle{}t be directly assigned to one certain layer can be found here.\hypertarget{md__2github_2workspace_2Architecture_autotoc_md10}{}\doxysection{\texorpdfstring{➕ Other relevant files}{➕ Other relevant files}}\label{md__2github_2workspace_2Architecture_autotoc_md10}
